## 🎯 Project Overview

An end-to-end AI system that:

* Trains an **LSTM neural network** to predict the next word in a text sequence
* Deploys the model as a **REST API** using **FastAPI**
* Provides real-time predictions via a `/predict` endpoint

**Task:** Text Prediction (Next Word Prediction)

**Deployment:** FastAPI with Swagger UI

**Dataset:** Project Gutenberg — *Alice's Adventures in Wonderland*

| Field                  | Details                                                                       |

| **Type**         | Plain text (public domain)                                                    |
| **Size**         | ~26,000 words                                                                 |
| **Description**  | Classic English narrative text — ideal for word-level LSTM sequence modeling |

### Preprocessing Steps:

1. Download raw text via HTTP request to Project Gutenberg
2. Strip Gutenberg header/footer metadata
3. Lowercase all text
4. Remove punctuation, digits, special characters (keep only a-z and spaces)
5. Tokenize into words using Keras `Tokenizer`
6. Build word → integer vocabulary mapping
7. Generate sliding window sequences of length `SEQ_LEN=10`
8. One-hot encode target words
9. Split 80% train / 20% validation

## 🧮 LSTM Mathematical Model

### Gates and State Equations

At each time step $t$, the LSTM receives input $x_t$, previous hidden state $h_{t-1}$, and previous cell state $C_{t-1}$.

#### Forget Gate — *What to erase*

$$
f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)
$$

#### Input Gate — *What new info to store*

$$
i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)
$$

$$
\tilde{C} *t = \tanh(W_C \cdot [h* {t-1}, x_t] + b_C)
$$

#### Cell State Update — *Update memory*

$$
C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t
$$

#### Output Gate — *What to output*

$$
o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)
$$

$$
h_t = o_t \odot \tanh(C_t)
$$

### Sequence Learning

* Each word embedding is fed as $x_t$ one at a time
* Cell state $C_t$ = long-term memory (preserved across many steps)
* Hidden state $h_t$ = short-term memory (passed between adjacent steps)
* Final $h_t$ fed to Dense(Softmax) → probability over vocabulary

## 📓 Running the Model (Colab)

1. Open `LSTM_TextPrediction_Colab.ipynb` in [Google Colab](https://colab.research.google.com/)
2. Run all cells top to bottom (`Runtime → Run all`)
3. The notebook will:
   * Download the dataset automatically
   * Preprocess and tokenize text
   * Train the LSTM model (~50 epochs with early stopping)
   * Save `lstm_model.h5`, `tokenizer.pkl`, `model_config.json`
   * Auto-download these files to your computer
4. Move the downloaded files into the `fastapi_app/` folder

## 🚀 Running the FastAPI Server

```bash
# Navigate to the fastapi_app folder
cd fastapi_app

# Start the server
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

You should see:

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     ✅ Tokenizer loaded. Vocabulary size: XXXX
INFO:     ✅ LSTM model loaded from 'lstm_model.h5'.
```

### Access Swagger UI

Open your browser and go to:

```
http://localhost:8000/docs
```

### `POST /predict` — Predict Next Word

**Request:**

```json
{
  "text": "alice was beginning to get very tired of sitting by",
  "top_k": 5,
  "temperature": 1.0
}
```

**Response:**

```json
{
  "input_text": "alice was beginning to get very tired of sitting by",
  "cleaned_input": "alice was beginning to get very tired of sitting by",
  "predicted_word": "her",
  "top_candidates": [
    {"word": "her",   "probability": 0.4521, "percentage": "45.21%"},
    {"word": "the",   "probability": 0.1832, "percentage": "18.32%"},
    {"word": "a",     "probability": 0.0921, "percentage": "9.21%"},
```
